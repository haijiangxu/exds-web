

## 🖥️ 零售套餐管理页面 UI 需求方案

以下是“零售套餐管理”页面的UI设计建议，旨在清晰、高效地配置上述复杂规则。

**页面名称：** 零售套餐管理

### 1. 主视图（套餐列表页）

**布局：** 顶部为操作栏，下方为套餐列表。

* **操作栏：**
    * **[ 搜索框 ]** （输入套餐名称关键字）
    * **[ 筛选器 - 下拉 ]** （按“套餐类型”筛选、按“状态”筛选）
    * **[ 按钮 ] + 新建零售套餐**
* **套餐列表（表格）：**
    * **套餐名称**
    * **套餐类型**（标签：分时段 / 不分时段）
    * **定价模式**（标签：固定+联动 / 价差分成）
    * **附加套餐**（标签：含绿色电力 / 含封顶价格）
    * **状态**（标签：草稿 / 生效 / 归档）
    * **创建时间**
    * **操作**（[ 编辑 ] [ 复制 ] [ 归档 ] [ 查看详情 ]）

#### 💡 UI 设计特别提示 (校验逻辑)

* **分时时段校验**：在“基本信息”标签页中，如果选择“分时段”，应有专门的模块配置时段划分（本文档未提供463号文的具体时段，故未画出）。该模块必须实时校验“谷总时长 $\ge$ 峰总时长”和“平总时长 $\ge$ 11小时” 。
* **浮动比例校验**：在“定价模式”标签页，当用户输入“自定义价格”后，系统应**实时计算**峰/平、谷/平、深谷/平的比例。如果比例不符合463号文（如文档第三个示例），应**立即显示一个警告信息**：“当前自定义价格比例不满足463号文要求，结算时将自动调整为标准比例 (1.6:1:0.4:0.3)。” 。
